# 控制台导览

OpenClaw Guard 的工作台不是把每个能力平铺成一面仪表盘，而是尽量让你先知道“现在能不能用”和“下一步去哪里”。

## 首页：先回答 3 个问题

首页首屏只聚焦三件事：

1. 当前可用性：Guard Web、OpenClaw、Gateway、关键服务是否就绪
2. 建议动作：现在最推荐先做哪一步
3. 风险提示：有没有必须优先处理的阻塞、异常或安全问题

如果你第一次用 Guard，建议不要先在侧边栏里逐页点，而是先把首页推荐动作走完。

## 核心控制台

### 运维

看服务是否正在运行、端口和路径是否正确、常见操作能否直接执行。

适合什么时候来：

- Guard Web 起不来
- 需要重启、停止或确认后台状态
- 想检查实际运行的是哪一个实例

### OpenClaw

这里负责 OpenClaw 本体的生命周期：

- 安装 / 修复
- 更新到推荐版本
- 版本回退
- 卸载
- 安装来源与更新策略诊断

首屏默认只给主动作，更复杂的能力放到高级区。

### 渠道

这里负责官方渠道的启用、配置和健康状态。首发版优先做“稳定可用”，不把自定义插件体系放在默认路径里。

### 模型

这里是模型配置入口。推荐先完成：

- 至少一个 Provider
- 一个明确的主模型
- 基本可用性测试

### 安全

安全页不只是堆开关，而是拆成 3 类更容易理解的内容：

- 安全检查
- 权限模式
- 主机加固

建议把它当成“边界与风险控制”页面，而不是高级配置仓库。

### 备份与恢复

这是 Guard 的核心卖点之一。默认优先用产品语言表达：

- 保存现在
- 查看恢复点时间线
- 回到某个状态
- 恢复后继续往前走

高级 Git 仍然在，但默认不抢镜。

## 工作区与角色

### 角色

统一查看和管理角色相关资产，适合关注行为模板、角色设定和长期维护。

### 文件

这里承接工作区里的核心 Markdown 资产，也包含记忆模式。它不是普通文件浏览器，而是更偏“工作区资产管理”。

### 搜索

适合在工作区、记忆和配置里快速找信息，而不是逐个目录翻找。

## 运行与排查

### 会话

观察当前运行中的会话状态、关键上下文和必要的用量信息。这里的“用量估算”不会被包装成账单能力。

### 日志

当你需要看最近发生了什么，这里是最直接的入口。推荐和首页、运维页结合使用，而不是单独从日志开始。

### 通知

把提醒和时间线合在一个页面里。它的目标不是做成消息中心，而是帮你快速知道“最近有哪些值得注意的变化”。

## 自动化

### Cron

这是面向更熟悉系统的用户的能力，用来管理周期任务或后台自动化。首发阶段不建议把它当成第一入口。
