## 第 2 页 · 这是一类典型的企业接入项目

企业微信和 Telegram / Discord 最大的不同，不在“消息能不能发”，而在：

- 需要管理员权限
- 回调地址必须可达
- 可见范围、应用权限、消息接收往往一起影响结果
- 很多团队会用课程配套或私有适配器，不一定是同一套字段

### 所以这页课的重点是

```text
平台应用创建 → 回调配置 → 适配器配置 → config validate → gateway restart → channels status --probe → 真机验证
```

> ⚠️ 如果你用的是课程配套 / 企业私有企业微信适配器，字段名和渠道 id 以那份 README 为准。

## 第 3.5 页 · 讲师串场话术

- **开场认知**：企业微信最大的难点通常不是技术，而是管理员权限、可见范围和回调可达性。
- **实操前提醒**：这节要提醒学员把“应用创建成功”和“真实用户看得到、收得到、回得出”分开理解。
- **卡点转场**：如果应用能建但消息不回，优先检查可见范围、Secret、Token、回调地址和 `openclaw channels status --probe`。
- **复盘收口**：企业微信 Lab 跑通后，学员基本就具备了交付企业内部 AI 助手项目的第一层实战感。

## 第 4 页 · 第一步：创建企业微信应用

### 操作步骤

1. 打开 [企业微信管理后台](https://work.weixin.qq.com)
2. 进入“应用管理” → “自建” → “创建应用”
3. 记录关键凭证：
   - `CorpId`
   - `AgentId`
   - `Secret`
   - `Token`
   - `EncodingAESKey`
4. 配置消息接收地址
5. 设置应用可见范围
6. 开启 API 接收消息

### 这一步最容易漏什么

- 应用创建了，但**你本人不在可见范围**
- 回调地址填了，但外网打不到
- Token / AESKey 和适配器里填的不一致

---

## 第 5 页 · 第二步：把企业微信适配器接进 OpenClaw

### 推荐顺序

1. 先确认你使用的适配器名称和渠道 id
2. 看适配器 README 是否支持 CLI 向导
3. 不支持的话，再写 `channels.<channel-id>` 配置

### 一个常见示意结构

```json5
{
  channels: {
    wecom: {
      enabled: true,
      corpId: "企业ID",
      agentId: "应用AgentId",
      secret: "应用Secret",
      token: "回调Token",
      encodingAESKey: "回调EncodingAESKey"
    }
  }
}
```

> ⚠️ 这里只是示意。你的实际适配器可能不是 `wecom`，字段名也可能不同。

### 配完以后立刻跑

```bash
openclaw config validate
openclaw gateway restart
openclaw channels status --probe
```

---

## 第 6 页 · 第三步：验证真实消息链路

### 测试步骤

1. 在手机端或桌面端打开你创建的企业微信应用
2. 发送：`你好`
3. 再发：`请介绍一下你自己`
4. 最后测一个真实任务：`帮我总结今天的待办`

### ✅ 成功标志

- 应用能收到并回复消息
- 不是只回第一条，后续也稳定
- 回复延迟在可接受范围内

> 如果第一条都收不到，优先查回调和可见范围；如果收到了但不回，优先查适配器和网关日志。

---

## 第 7 页 · 故障排查速查

| 问题 | 先查什么 |
|------|----------|
| 回调验证失败 | 外网是否能访问；Token / AESKey 是否一致 |
| 应用创建了但自己看不到 | 可见范围是否包含你的账号 |
| 能收到事件但不回复 | Secret 是否正确；Gateway 是否已重启；适配器是否正常加载 |
| 回复被截断 | 企业微信消息长度限制；让 AI 输出更短更结构化 |
| 群里不响应 | 群接入方式、@ 触发策略、适配器是否支持群消息 |
| 改了配置没生效 | 是否忘了 `openclaw gateway restart` |

### 需要更细定位时

```bash
openclaw logs --follow
```

---

## 第 8 页 · 企业交付里的三个习惯

1. **先做测试应用，再上正式应用**
   不要拿正式企业应用直接调试。

2. **先跑单聊，再跑群聊**
   单聊稳定以后，再处理群消息、@ 规则、权限边界。

3. **把“哪套适配器 + 哪套字段 + 哪个回调地址”写进交付文档**
   企业微信最怕交接时信息断层。
