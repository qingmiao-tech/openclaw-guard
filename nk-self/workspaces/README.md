# 南柯个人工作区

本目录参考 `customer/openclaw-biz-team/workspaces` 的结构，改造成南柯个人版工作区。

## 命名规则

- Agent `id` 统一采用 `nk-nan*` 风格，兼顾技术稳定性与个人气质。
- 展示名统一采用“南 + 意象词 + 功能位”结构。
- 工作区目录与 Agent `id` 保持一致，全部使用 `nk-*` 前缀。

- `nk-nanxin` → `nk-nanxin` → `南心-总管（人生中枢 / 任务总控）`
- `nk-nangong` → `nk-nangong` → `南工-研造（产品开发 / AI 培训）`

- `nk-nanqi` → `nk-nanqi` → `南岐-杏林（中医学习 / 经典研读）`
- `nk-nanhe` → `nk-nanhe` → `南和-家宁（备孕支持 / 家庭健康）`
- `nk-nanchuang` → `nk-nanchuang` → `南窗-读伴（读书导师 / 知识沉淀）`

## 目录说明

- 每个子目录下的 `SOUL.md`：定义角色人格、边界、语气与核心职责。
- 每个子目录下的 `USER.md`：定义“这个角色视角下的南柯画像与关注重点”。
- 每个子目录下的 `AGENTS.md`：定义执行协议、开场风格、输出标准和协作方式。

## 使用建议

- 默认由 `nk-nanxin` 承接模糊任务。
- 明确任务可直达专业 Agent。
- 高风险主题（医疗、备孕、用药、诊断）只做学习辅助和信息整理，不替代专业医生。
