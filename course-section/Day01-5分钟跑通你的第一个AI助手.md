## 第 3 页 · OpenClaw 是什么？

### 一句话定义

OpenClaw 是一个运行在**你自己电脑上**的 AI 助手框架

- 🔓 **开源免费**：由独立基金会治理，社区驱动，持续更新
- 🏠 **数据本地化**：对话和数据不会被第三方收集
- 🎨 **可定制**：给 AI 设定性格、安装技能、对接消息渠道
- 🌍 **全球热门**：GitHub 180,000+ 星标，2026 年增长最快的开源 AI 项目

### 它能做什么？

- 💬 智能聊天
- 🛠️ 执行任务（创建文件、搜索信息）
- 🤖 自动化日常工作

### 项目背景

OpenClaw 由奥地利开发者 Peter Steinberger 于 2025 年底创建，最初名为 Clawdbot，后更名为 Moltbot，最终定名 OpenClaw。2026 年 1 月公开发布后迅速走红，一周内获得 10 万+ GitHub 星标。2026 年 2 月，创始人加入 OpenAI 专注多 Agent 系统研发，OpenClaw 项目转为独立基金会运营，保持开源和独立。

### 支持的 AI 模型

OpenClaw 支持多种主流 AI 模型，你可以根据需求和预算选择：

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| Claude（Anthropic） | 安全、擅长长文本 | 通用推荐 |
| GPT-4/GPT-5（OpenAI） | 能力全面 | 通用推荐 |
| Gemini 3.1（Google） | 多模态、大上下文 | 图片理解、长文档 |
| Hugging Face 模型 | 开源、可本地运行 | 隐私优先、离线场景 |

> 💡 架构原理将在 Day 3 深入讲解，今天先跑起来！

## 第 5 页 · 环境准备

### 检查 Node.js

打开终端，输入以下命令：

```bash
node -v    # 查看 Node.js 版本，需要 ≥ 22
npm -v     # 查看 npm 版本
```

### 没有 Node.js？

1. 前往 https://nodejs.org
2. 下载 **LTS 版本**
3. 安装时确保勾选 "Add to PATH"

> ⚠️ 当前 OpenClaw 运行时要求已经提升，课程统一按 **Node.js 22+** 准备环境。

---

## 第 6 页 · 安装 OpenClaw

### 一条命令搞定

```bash
npm install -g openclaw@latest
```

### 验证安装

```bash
openclaw --version
# 输出类似：openclaw 2026.x.x

openclaw config validate
# 输出类似：Config valid
```

### 如果你想再多做一步

```bash
openclaw gateway health
```

> 💡 如果这里提示 Gateway 不可达，不代表你没装好，通常只是还没完成引导或网关还没启动。

### 常见问题

| 问题 | 解决方案 |
|------|----------|
| 权限错误（EACCES） | macOS/Linux 加 `sudo`，或配置 npm 全局目录 |
| "命令未找到" | Node.js 未安装或未加入 PATH |

## 第 7 页 · 初始配置（onboard 向导）

### 启动向导

```bash
openclaw onboard
```

### 配置步骤

1. 🤖 选择 AI 模型（推荐 Claude 或 GPT）
2. 🔑 输入 API Key（从 AI 服务商获取）
3. ✅ 完成剩余配置项

### API Key 获取方式

- **OpenAI**：https://platform.openai.com → API Keys
- **Anthropic**：https://console.anthropic.com → API Keys

> 💡 大多数服务商提供免费试用额度，足够完成课程学习

### 做完引导后，立刻跑一次自检

```bash
openclaw config validate
openclaw gateway health
```

> 如果 `config validate` 通过，但 `gateway health` 失败，优先检查是否完成了引导、API Key 是否可用、网络是否可达。

## 第 11 页 · 实操任务 1：安装与配置

### 操作步骤

1. 打开终端
2. `node -v` → 确认 Node.js

- `openclaw --version` 输出版本号

## 第 14 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| npm 安装报权限错误 | 加 `sudo` 或配置 npm prefix |
| node -v 命令未找到 | 重新安装 Node.js，勾选 Add to PATH |
| `openclaw config validate` 失败 | 先看报错字段；再运行 `openclaw doctor --fix` 修复可迁移问题 |
| onboard 网络错误 | 检查网络；配置代理 |
| API Key 无效 | 重新运行 `openclaw onboard` |
| TUI 启动失败 | 确认 onboard 已完成；使用现代终端 |
| 发送消息无回复 | 检查网络；确认 API Key 余额 |

## 第 15 页 · 🔒 安全提示

- **API Key 是你的"钥匙"**：不要分享给他人，不要上传到公开仓库
- **API Key 有费用**：关注服务商账户余额，避免意外扣费
- **本地数据安全**：对话数据存储在本地，保护好电脑安全

## 第 17 页 · 社区实战案例：零基础也能搞定

### 不会写代码？腾讯云一键部署了解一下

其实还有一条更简单的路径——腾讯云已经把 OpenClaw 做成了**一键部署模板**，买服务器时勾选即可，全程网页操作：

1. 打开腾讯云轻量应用服务器页面
2. 应用模板选择 **OpenClaw（Clawdbot）**
3. 最低 ¥99/年（约 ¥8.25/月），2 核 2G 足够
4. 买完即装好，进入"应用管理"面板配置 API Key 和聊天渠道

### 零基础用户的真实体验

大量非程序员用户——运营、产品经理、自媒体人、独立创业者——都在使用 OpenClaw。他们的反馈是：

- 全程不需要打开终端，网页上点几下就搞定
- 配置 AI 模型只需要粘贴一个 API Key
- 接入 QQ/企微/飞书等聊天平台，也是在面板里填凭证
