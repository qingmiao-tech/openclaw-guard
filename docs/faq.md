# FAQ

## Guard 和 OpenClaw 是什么关系？

OpenClaw 是主体运行时，Guard 是围绕它做安装、运维、安全、防误触和恢复的一层控制台。

## 恢复之后，后续版本会怎么继续？

Guard 默认不会强制你理解分支管理。恢复会在当前主线上追加一个新的 restore commit，后续版本继续接在这条主线上。

## 我完全不懂 Git，可以用备份与恢复吗？

可以。普通用户默认看到的是恢复中心，不需要先理解 raw Git 术语；高级 Git 只是给技术用户的专家视图。

## 为什么首页不直接把所有功能都展示出来？

因为首发阶段最重要的是让用户“第一次就知道先做什么”。日志、Cron、更多高级运维入口仍然保留，但不会抢主舞台。

## 渠道页为什么不单独保留一个“自定义飞书”菜单？

公开版优先考虑通用性。项目定制插件未来会以“官方渠道 + 自定义扩展”的方式规划，但不会在首发阶段当成默认能力承诺。

## 成本功能去哪了？

当前成本能力更接近“使用估算”，而不是账单数据，所以不会作为首发主功能突出展示。

## 如果工作区里有绘画、导出这类大目录怎么办？

首发阶段会先用“保护范围摘要”告诉你哪些内容值得优先保护。后续版本会继续加强对绘画、导出和新 Workspace 的识别与提醒。
