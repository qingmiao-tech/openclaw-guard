## 第 3 页 · 为什么要自己写 Skill？

### 三个理由

1. 🎯 社区技能**无法满足**所有个性化需求
2. 📦 将工作经验和流程**沉淀**为可复用的技能
3. 🚀 让 AI 按照**你的方式**做事

### 三种沉淀方式

| 类型 | 说明 | 难度 |
|------|------|------|
| 流程型（SOP） | 标准操作流程 | ⭐ 最容易 |
| 工具型 | 自动化脚本封装 | ⭐⭐ |
| 判断型 | 决策规则和逻辑 | ⭐⭐⭐ |

> 💡 建议从**流程型**起步，最容易上手！

## 第 4 页 · Skill 文件结构

### 目录结构

```
skills/
└── my-skill/
    ├── skill.md       # 技能定义文件（核心）
    └── README.md      # 技能说明（可选）
```

### skill.md 核心内容

| 字段 | 说明 |
|------|------|
| 技能名称 | 简洁明了，说明用途 |
| 技能描述 | 告诉 Agent 什么时候该用它 |
| 执行步骤 | 自然语言描述的操作流程 |
| 输入参数 | 技能需要的输入信息 |
| 输出格式 | 执行后的输出格式要求 |

---

## 第 5 页 · Skill 编写规范

### 编写技巧

- ✅ 步骤要**具体、可执行**
- ✅ 描述要**明确**，避免模糊
- ✅ 考虑**异常情况**处理
- ❌ 不要写太长，保持简洁

### 好的描述 vs 差的描述

| 差的描述 | 好的描述 |
|----------|----------|
| "搜索新闻" | "使用 Web 搜索工具，搜索最近 24 小时内的 AI 领域热点新闻，至少找到 5 条" |
| "整理内容" | "将搜索到的新闻按重要性排序，每条提取标题、来源和一句话摘要" |
| "输出结果" | "以 Markdown 列表格式输出，每条包含：序号、标题、来源、摘要" |

## 第 6 页 · 开发案例：每日新闻摘要 Skill

### 需求分析

- 🎯 目标：每天自动获取 AI 领域热点新闻，生成中文摘要
- 📥 输入：无（自动搜索）
- 📤 输出：结构化的中文新闻摘要

### 设计思路

```
Web 搜索获取新闻 → 筛选 AI 相关内容 → 生成中文摘要 → 固定格式输出
```

---

## 第 7 页 · skill.md 编写示例

```markdown
# 每日 AI 新闻摘要

## 描述
当用户要求获取 AI 新闻、科技热点或每日资讯时，使用此技能。

## 执行步骤
1. 使用 Web 搜索工具，搜索"AI 人工智能 最新新闻"
   和"artificial intelligence news today"
2. 从搜索结果中筛选最近 24 小时内的热点内容
3. 选取最重要的 5-8 条新闻
4. 对每条新闻提取：标题、来源、一句话中文摘要
5. 按重要性排序

## 输出格式
📰 AI 热点日报 | {日期}

1. **{标题}** - {来源}
   {一句话摘要}

2. ...（重复以上格式）

---
以上资讯由 AI 助手自动整理，仅供参考。
```

## 第 8 页 · 调试方法

### 调试三步法

1. **查看日志**：观察 AI 是否识别并调用了你的 Skill
   ```bash
   pm2 logs openclaw-gw --lines 50
   ```
2. **逐步测试**：先测试单个步骤，再测试完整流程
3. **迭代优化**：根据输出效果调整 skill.md

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| AI 不调用你的 Skill | 描述不够明确 | 优化技能描述，加入触发关键词 |
| 执行结果不符预期 | 步骤描述不够具体 | 细化每个步骤的操作指令 |
| Skill 文件找不到 | 路径错误 | 确认在 skills/ 目录下 |

## 第 10 页 · 实操任务 1：创建自定义 Skill

### 操作步骤

1. `cd skills/` → 进入技能目录
2. `mkdir daily-news && cd daily-news` → 创建技能文件夹
3. `nano skill.md` → 编写技能定义
4. 按照编写规范填写名称、描述、步骤、输出格式
5. 保存文件
6. `pm2 restart openclaw-gw` → 重启 Gateway
7. `openclaw tui` → 测试："帮我生成今天的 AI 新闻摘要"

### ✅ 成功标志

- AI 识别并调用你的自定义 Skill
- 按照定义的步骤生成新闻摘要

---

## 第 11 页 · 实操任务 2：迭代优化

### 操作步骤

1. 分析第一次测试的输出，找出不满意的地方
2. `nano skill.md` → 调整步骤描述或输出格式
3. `pm2 restart openclaw-gw` → 重启并重新测试
4. 重复 2-3 次迭代，直到输出满意

### 优化方向

- 输出格式不够美观 → 调整格式模板
- 新闻不够相关 → 优化搜索关键词
- 摘要质量不高 → 细化摘要要求

### ✅ 成功标志

- 经过迭代后，Skill 输出质量明显提升

---

## 第 12 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| AI 没有调用 Skill | 检查描述是否明确；确认路径正确；重启 Gateway |
| 执行结果不符预期 | 细化步骤描述；加入更明确的指令 |
| 多次迭代效果仍不理想 | 将复杂 Skill 拆分为多个简单步骤 |
| Skill 调用报错 | 查看日志定位具体错误 |

## 第 13 页 · 🔒 安全提示

- **Skill 权限控制**：注意限制 Skill 的操作范围，避免执行危险操作
- **不要硬编码敏感信息**：API Key、密码等应通过环境变量引用
- **测试环境隔离**：新 Skill 先在测试环境验证，避免影响正式运行
