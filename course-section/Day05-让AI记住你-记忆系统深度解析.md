## 第 4 页 · 三层记忆架构

```
┌─────────────────────────────────────┐
│         核心记忆 (Core Memory)        │  ← 你的"名片"，始终可用
│   姓名、职业、时区、核心偏好            │
├─────────────────────────────────────┤
│        长期记忆 (Archival Memory)      │  ← 你的"笔记本"，按需检索
│   项目记录、会议纪要、学习笔记          │
├─────────────────────────────────────┤
│        短期记忆 (Conversation Memory)  │  ← 你的"工作台"，当前对话
│   当前对话中提到的任务和临时信息         │
└─────────────────────────────────────┘
```

## 第 5 页 · 核心记忆（Core Memory）

### 你的"名片和个人简介"

- 存储**最重要**的用户信息和关键事实
- **始终可用**，每次对话都会被 Agent 读取
- 容量有限，只放最关键的信息

### 典型内容

- 用户姓名、称呼
- 职业、工作领域
- 时区、所在地
- 核心偏好（如"喜欢简洁回复"）

> 💡 核心记忆就像 AI 随身携带的小卡片，随时可以翻看

---

## 第 6 页 · 长期记忆（Archival Memory）

### 你的"笔记本和档案柜"

- 存储**大量**历史信息
- **不会每次都全部加载**，Agent 需要时主动搜索
- 容量大，适合存储详细记录

### 典型内容

- 项目进展记录
- 会议纪要
- 学习笔记
- 重要事件和决策

> 💡 长期记忆就像图书馆——信息很多，需要时去查找

---

## 第 7 页 · 短期记忆（Conversation Memory）

### 你的"工作台上正在处理的文件"

- 当前对话的**上下文**
- 对话结束后可**归档**到长期记忆
- 让 AI 在一次对话中保持连贯

### 典型内容

- 当前对话中提到的任务
- 临时信息和中间结果
- 对话中的上下文线索

> 💡 短期记忆让 AI 不会"前一句话就忘了"

---

## 第 8 页 · 记忆文件存储结构

### 目录结构

```
memory/
├── core/              # 核心记忆文件
│   └── user-info.md   # 用户关键信息
├── archival/          # 长期记忆文件
│   ├── project-a.md   # 项目记录
│   └── notes.md       # 学习笔记
└── conversations/     # 对话记录
    └── 2024-01-15.md  # 按日期归档
```

### 文件格式

- Markdown 或 JSON 格式
- **可直接查看和编辑**——记忆对你完全透明

## 第 9 页 · 记忆操作：查看

### 通过命令行查看

```bash
# 查看核心记忆
cat memory/core/*.md

# 查看长期记忆列表
ls memory/archival/

# 查看某条长期记忆的内容
cat memory/archival/project-a.md
```

### 通过对话查看

直接问 AI：

- "你记得我是谁吗？"
- "你知道我在做什么项目吗？"
- "你记得我上次说了什么？"

## 第 10 页 · 记忆操作：编辑与清除

### 编辑记忆

```bash
# 直接编辑记忆文件
nano memory/core/user-info.md

# 通过对话让 AI 记住
# → "请记住我的邮箱是 xxx@example.com"
```

### 清除记忆

```bash
# 清除特定记忆文件
rm memory/archival/old-notes.md

# 清除所有记忆（谨慎！）
rm -rf memory/*

# 通过对话让 AI 忘记
# → "请忘记我之前说的 xxx"
```

> ⚠️ 删除记忆文件后无法恢复，建议先备份：`cp -r memory/ memory-backup/`

## 第 15 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| AI 在新会话中不记得信息 | 检查 memory/ 目录是否有文件；明确说"请记住这个" |
| memory/ 目录为空 | 正常现象，AI 还没主动写入记忆 |
| 手动添加的记忆 AI 读不到 | 确认文件路径正确；确认已重启 Gateway |
| 记忆文件越来越大 | 手动清理过时的记忆文件 |

## 第 18 页 · 社区实战案例：记忆系统的实战调优

> 💬 来源：社区 OpenClaw 2.6 调教实录

### 记忆丢失的真实事故

实际使用中你可能会遇到这种情况：AI 助手在长对话后会"失忆"——之前记住的用户偏好、项目进度突然消失。根因是 OpenClaw 的上下文压缩（compaction）机制在压缩时把重要记忆挤掉了。

### 三个关键配置

**① 记忆刷盘保护**

```json
"compaction": {
  "mode": "safeguard",
  "memoryFlush": true
}
```

压缩前自动把关键信息写入 MEMORY.md，确保跨会话记忆不丢失。

**② 上下文自动裁剪**

```json
"contextPruning": {
  "mode": "cache-ttl",
  "ttl": "5m"
}
```

超过 5 分钟的工具输出自动裁剪，上下文保持精简，token 费用降低 20-30%。注意：这只影响发给 AI 的上下文，磁盘上的完整历史不受影响。

**③ 会话自然过期**

```json
"session": {
  "resetMode": "idle",
  "idleMinutes": 240
}
```

4 小时没聊天才重置会话，比固定时间重置更自然——深夜聊到 2 点也不会被打断。

### 进阶：用 QMD 替代默认记忆搜索

社区推荐使用 QMD（本地搜索工具），用 BM25 + 向量 + 重排序三重机制检索记忆文件：

- 零 API 费用，零延迟，数据不出机器
- 只拉最相关的 2-3 句话进上下文，token 节省 60-97%
- Markdown 文件仍然是唯一的"真相来源"

> 💡 没有记忆的 AI 助手就是个高级复读机。记忆系统的调优，直接决定了 AI 能不能"越用越懂你"。
