## 第 2.5 页 · 讲师提示语（可直接口播）

- **开场认知**：内容自动化不是让 AI 取代创作者，而是先帮你取代那些重复、耗时、低创造性的环节。
- **实操前提醒**：顺序不要乱——先学风格，再出初稿，最后再把它串成自动化管线。
- **卡点转场**：文章质量不稳定时，先补样本、补要求、补结构，不要第一时间去换更贵的模型。
- **复盘收口**：今天结束时，学员最好能拿到一个“能持续产出选题或初稿”的最小工作流。

> 🎯 正确定位：AI 是"写作助手"而非"写作替代"

## 第 3.5 页 · 自动化边界：自动到初稿，不自动到发布

这页建议课堂上明确说透，不然后面大家最容易“为了自动化而自动化”。

| 环节 | 是否建议自动化 | 原因 |
|------|------|------|
| 热点收集 | ✅ 建议自动 | 重复、费时间、规则相对明确 |
| 选题候选生成 | ✅ 建议自动 | AI 擅长发散与聚合 |
| 大纲生成 | ✅ 建议自动 | 结构化输出稳定 |
| 初稿撰写 | ✅ 建议自动 | 提升速度最明显 |
| 观点定稿 | ⚠️ 人机协作 | 需要你的经验、判断和立场 |
| 最终发布 | ❌ 不建议全自动 | 涉及事实、版权、品牌风险 |

一句话记忆：

> **内容创作自动化的目标不是“无人值守发稿”，而是“把你从机械重复里解放出来”。**

## 第 5 页 · 项目实战：AI 自动写文章

### 项目背景

让 AI 每天自动收集热点、生成文章初稿

### 实现架构

```
AI 学习写作风格（一次性）
    ↓
风格沉淀到 `memory.md / USER.md`
    ↓
编写"内容创作" `SKILL.md`
    ↓
Cron 定时执行选题推送
    ↓
按需生成文章初稿
    ↓
人工审核后发布
```

### 涉及能力

Skill + Cron + Memory + Web 搜索 + 文件操作

---

## 第 6 页 · 第一步：让 AI 学习写作风格

### 操作方法

1. 准备 2-3 篇参考文章
2. 发送给 AI：

> "请分析以下文章的写作风格特征，包括标题风格、段落结构、语气、用词习惯"

3. 让 AI 记住风格：

> "请记住这些写作风格特征"

4. 验证：让 AI 按学到的风格写一段短文

### 更稳的做法：把风格落成一张“风格卡”

不要只停留在聊天里，建议把风格总结成 8-12 条可复用规则，写进工作区里的长期文件：

- `memory.md`：长期有效的写作偏好、禁写项、表达边界
- `USER.md`：你是谁、你的受众是谁、你的品牌定位是什么

这样做的好处是：

- 新会话也能更稳定继承
- 以后改风格，只改一页就行
- Day 12 做毕业项目时，能直接复用这套风格资产

---

## 第 7 页 · 第二步：编写内容创作 Skill

### `SKILL.md` 示例

```markdown
# 每日选题推送

## 描述
当用户要求推送选题或内容灵感时使用此技能。

## 执行步骤
1. 使用 Web 搜索获取用户关注领域的最新热点
2. 筛选出 3-5 个有写作价值的选题
3. 对每个选题给出：
   - 标题建议
   - 核心观点
   - 目标读者
   - 预估字数
4. 按推荐优先级排序输出
```

---

## 第 7.5 页 · 内容创作 Skill 的最小拆分法

课堂上不要一开始就做“全自动写作大而全 Skill”，更稳的做法是拆成两段：

1. **选题 Skill**：负责搜集、筛选、生成 3-5 个候选题
2. **写稿 Skill**：负责根据指定题目生成大纲和初稿

这么拆的好处：

- 好排查：你能知道是“题选歪了”还是“稿写歪了”
- 好迭代：改选题逻辑时，不会连写稿逻辑一起打坏
- 好控风险：可以把“人工选题确认”放在中间，避免自动跑偏

## 第 8 页 · 第三步：配置自动化

### 配置 Cron 定时任务

```bash
# 每天下午 2 点执行选题推送
openclaw cron add \
  --name "Daily content topics" \
  --cron "0 14 * * *" \
  --session isolated \
  --message "执行每日选题推送任务：收集与你的主题相关的热点，输出 3-5 个可写选题，每个包含标题建议、核心观点和目标读者"
```

### 按需生成文章

选择一个选题后，告诉 AI：

> "请按照我的写作风格，围绕选题 xxx 写一篇 1500 字的文章初稿"

## 第 9.5 页 · 内容资产放哪里，后面最省心

建议把内容创作相关的资料固定放在工作区里，形成一个可复用的创作桌面：

```text
workspace/
├── USER.md                 # 你的身份、受众、账号定位
├── memory.md               # 长期风格、禁写项、常用表达
├── references/             # 参考文章、素材、案例
├── drafts/                 # AI 生成的草稿
└── skills/
    └── content-topic/
        └── SKILL.md
```

这比“全部靠聊天记住”稳很多，也更方便 Day 12 做项目验收。

## 第 11 页 · 实操任务 1：学习写作风格

### 操作步骤

1. 准备 2-3 篇参考文章
2. 发送给 AI 并要求分析风格特征

4. 把账号定位、目标读者补进 `USER.md`
5. 验证：让 AI 按学到的风格写一段短文

### ✅ 成功标志

- AI 能分析出写作风格特征
- 后续写作中能应用该风格
- 你手上已经有一份可维护的“风格卡”，而不是只停留在聊天记录里

---

## 第 12 页 · 实操任务 2：构建创作管线

### 操作步骤

1. 编写“每日选题推送” `SKILL.md`
2. `openclaw skills list --eligible` → 确认当前会话可见该 Skill
3. 配置 Cron：每天下午 2 点执行选题推送
4. 手动跑一次：`openclaw cron run <任务ID> --force`
5. 选择一个选题，让 AI 生成文章初稿
6. 审核初稿，提出修改意见
7. 将最终稿件保存到 `drafts/` 目录

### ✅ 成功标志

- 每天自动推送选题建议
- 可按需生成文章初稿
- 你能明确区分“选题自动化”和“发布审核”两个阶段

## 第 13 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| AI 分析的风格不准确 | 提供更多参考文章；给出更具体的分析维度 |
| 文章质量不高 | 优化写稿 Skill；给出更详细的结构模板和禁写项 |
| 选题不够相关 | 优化搜索关键词；在记忆中明确关注领域 |
| 风格不一致 | 检查 `memory.md / USER.md` 是否真的写清了风格卡和账号定位 |
| 定时推送正常，但写稿总跑偏 | 说明“选题 Skill”和“写稿 Skill”没有拆开，先拆成两段再调 |

---

## 第 14 页 · 🔒 安全提示

- **内容版权**：AI 生成的内容可能与已有文章相似，发布前请人工审核原创性
- **信息准确性**：AI 可能生成不准确的信息，发布前务必核实关键事实
- **不建议直接发布**：AI 生成的是初稿，需要人工审核后再发布
- **来源留痕**：热点、数据、案例最好保留原始来源链接，方便复核

## 第 16 页 · 社区实战案例①：10 分钟写文章发 14 个平台

> 💬 来源：社区全流程拆解

### 从 3 小时到 10 分钟的效率革命

下面来看一个叫"墨笔"的内容 Agent 实战案例，实现了从选题到全平台发布的自动化：

**Before vs After**

| 环节 | 以前 | 现在 |
|------|------|------|
| 选题 | 30 分钟刷信息流 | Agent 每天 9 点推 5 个选题 |
| 写作 | 1-2 小时 | Agent 5 分钟出初稿，改 5 分钟 |
| 排版 | 30 分钟 | 自动推飞书，格式已排好 |
| 发布 | 30 分钟手动发各平台 | 字流一键发 14 个平台 |

**核心架构**

1. SOUL.md 定义 Agent 角色和边界
2. 风格文件（3000 字）定义写作规范——开头模式、标题公式、结构习惯
3. Cron 每天 9 点自动执行选题推送
4. Agent 收到确认后自动完成：素材收集 → 按风格写初稿 → 推送飞书

**关键原则**：绝不自动发布。AI 负责效率，人负责质量底线。

## 第 17 页 · 社区实战案例②：AI 做视频，播放量翻 10 倍

> 💬 来源：社区视频自动化实践

### 技术栈：OpenClaw + Remotion + 语音克隆

这个案例用 AI 全自动做视频号内容，3 天总播放 9018，单条最高 1595（之前平均不到 200）。

**四个关键零件**

| 零件 | 作用 | 成本 |
|------|------|------|
| OpenClaw | 多 Agent 调度中心 | 服务器费用 |
| Remotion | React 视频框架，代码生成视频 | 免费开源 |
| MiniMax TTS | 语音克隆，用自己的声音 | ～¥0.1/条 |
| Whisper | 提取逐句时间戳 | 本地免费 |

**完整流程**

```
Agent 推选题 → 选一个 → Agent 写旁白 → TTS 生成语音
→ Whisper 提取时间戳 → Remotion 渲染成片 → 确认发布
```

总耗时约 15 分钟，成本不到两毛钱。

**为什么 AI 做的反而更好？**

1. 更新频率上去了，算法喜欢活跃账号
2. 风格统一，辨识度高
3. 质量恒定，没有人工状态起伏

> 💡 视频 = 数据 + 模板 + 自动化。把视频从"项目"变成"工序"，是内容创作自动化的核心思路。
