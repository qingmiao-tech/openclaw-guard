# 课程文档本轮更新说明

> 更新时间：2026-03-12  
> 适用范围：`course-design.md` + `course-section/*`

---

## 一、这轮更新解决了什么

这一轮不是简单润色，而是把课程主设计稿与分日课件，整体同步到当前 OpenClaw 的教学与实操口径：

- 课程更贴近 **OpenClaw 2026.x** 的实际 CLI 与工作流
- 课程结构更符合“**先做再懂、理论穿插在实践里**”的授课原则
- 每个 Day / Lab 都补上了 **讲师可直接口播** 的串场话术
- 高影响模块的旧命令、旧术语、旧教学路径已同步修正

---

## 二、核心变化概览

### 1. 统一补齐讲师提示语

在 Day 1–14 主课件中，统一新增了四类讲师提示：

- 开场认知
- 实操前提醒
- 卡点转场
- 复盘收口

这样做的目的，是让课件不只是“给学员看的 PPT”，也能直接作为讲师带课的节奏稿。

### 2. 统一补齐渠道 Lab 串场话术

在 Lab A–F 中，统一增加了“讲师串场话术”，帮助课堂上快速讲清：

- 这个渠道为什么值得教
- 这一节最容易卡在哪里
- 出问题时优先排什么
- 跑通后对整体课程理解有什么帮助

### 3. Day 2 改为官方守护服务主路径

将 Day 2 从过去偏 `pm2` 的主路径，切换到当前更推荐的官方 CLI 路径：

- `openclaw onboard --install-daemon`
- `openclaw gateway install`
- `openclaw gateway start`
- `openclaw gateway status`
- `openclaw gateway restart`
- `openclaw gateway health`

其中：

- `pm2 / systemd` 仍然保留
- 但已经降级为“补充了解 / 融入存量环境”的口径
- 主线教学不再要求新手先学第三套进程管理方式

### 4. Day 7 Skill 开发调试口径升级

Skill 课现在更贴近真实排障流程，重点强调：

- `SKILL.md` 文件名和路径优先级
- 新会话验证的重要性
- `openclaw skills list --eligible` 判断“当前会话是否真的可见”
- `openclaw logs --follow` 观察真实调用情况

这样能明显减少“明明写了 Skill，但 AI 就是不调用”的课堂困惑。

### 5. Day 11 安全运维口径与当前 CLI 对齐

安全运维模块统一强化为一条更稳定的诊断闭环：

- `openclaw status`
- `openclaw gateway status`
- `openclaw config validate`
- `openclaw gateway health`
- `openclaw logs --follow`

同时修正了旧写法：

- `openclaw skill list` → `openclaw skills list`

### 6. Day 13 / Day 14 多 Agent 交付链路更清晰

多 Agent 相关内容进一步统一到当前教学重点：

- 明确 `agentId / accountId / sessionKey` 三层关系
- 修正 `USER.md` 等命名口径
- 多 profile 部署更贴近官方方式
- Day 14 更明确定位为“**交付课**”，强调可复制、可验收、可回退

### 7. 补充资料也变成可教学材料

`补充-OpenClaw热加载与重启机制详解.md` 已补充“讲师使用建议”，便于讲师在学员卡在：

- 改了工作区文件要不要重启
- 改了 `openclaw.json` 为什么没生效
- 改了渠道配置为什么 Bot 还不回

这些高频问题时，快速拿来辅助解释，而不是临场现编。

## 三、最值得关注的教学收益

从授课角度看，这轮更新最重要的价值有四点：

### 1. 课程更像“带着学员做项目”

不是先堆理论，再做实验；而是直接让学员在任务中建立理解。

### 2. 主设计稿与分日课件不再脱节

`course-design.md` 的教学思路，已经更系统地落到了 `course-section/*`。

### 3. 课堂排障口径更统一

以后讲课时，很多问题都可以统一回到这些动作：

- `config validate`
- `gateway restart`
- `gateway health`
- `channels status --probe`
- `logs --follow`

### 4. 讲师负担更轻

补齐口播提示后，讲课时不用再频繁临时组织：

- 开场怎么讲
- 学员卡住时怎么转场
- 这一节最后怎么收口
