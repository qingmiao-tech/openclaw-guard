# 备份与恢复

这是 Guard 最重要的卖点之一。

Guard 不希望普通用户先理解 Git，而是先理解这三件事：

- 保存现在
- 恢复到某个状态
- 恢复后还能继续往前走

## 默认视图：恢复中心

恢复中心会优先展示：

- 当前保护状态
- 下一步建议
- 立即保存
- 恢复点时间线
- 保护范围摘要

它尽量用人话告诉你：

- 现在有没有恢复点
- 最近一次保护有没有上云
- 当前是否还有没纳入保护的本地变化

## “保存现在”到底做了什么

它的产品语义是：

- 如果远端和认证已经准备好，会执行保存并同步到云端
- 如果远端还没准备好，会先保存在本机

所以它不是单纯“点一下 commit”，而是“为当前状态建立恢复点”。

## “回到这个状态”之后会怎样

Guard 默认采用非破坏式恢复：

- 不自动新建分支
- 不做强推
- 不改写公开历史
- 在当前主线上追加一个新的 restore commit

这样做的好处是：

- 用户不需要理解分支管理
- 历史不会被悄悄删除
- 恢复后的下一次保存可以继续往前走

## 恢复前的安全保护

如果当前工作区里还有未保存变更，Guard 会先尝试创建一个自动保护点，再继续恢复。

如果自动保护失败，恢复会被阻止，并提示你先处理当前变更。

## 高级 Git

技术用户仍然可以切到“高级 Git”视图，去看：

- 仓库连接
- 同步范围
- `.gitignore` 建议
- 认证与推送

普通用户如果只是想“保住现在、回到以前”，不需要先理解这些内容。

## 推荐实践

- 第一次进入时，先创建首个恢复点
- 每次要做高风险动作前，先保存现在
- 恢复后先验证系统是否正常，再继续后续变更
- 如果是本机恢复为主，后面再逐步补齐私有仓库连接和云端保护
