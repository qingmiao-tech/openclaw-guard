## 第 2 页 · 渠道简介

### 为什么选 QQ？

- 🇨🇳 国内最主流的即时通讯平台之一

- 🤖 支持 QQ 群机器人和频道机器人

### 适用场景

- 国内社群运营
- 好友互动
- QQ 频道智能助手

- ✅ Day 2 完成（OpenClaw 已在服务器上运行）

## 第 3 页 · 创建 QQ 机器人

### 操作步骤

1. 访问 [QQ 开放平台](https://q.qq.com)，QQ 账号登录
2. 完成**开发者实名认证**（需身份证信息）
3. 进入"应用管理" → "创建应用"
4. 选择应用类型（推荐"QQ 群机器人"或"QQ 频道机器人"）
5. 填写应用信息：名称、描述、图标
6. 获取凭证：
   - **App ID**：应用唯一标识
   - **App Secret**：应用密钥
   - **Token**：机器人验证令牌
7. 配置消息接收（Webhook 或 WebSocket）
8. **提交审核**（通常 1-3 个工作日）

> ⚠️ QQ 平台需要审核，请提前准备

---

## 第 4 页 · 配置 OpenClaw

### 编辑配置文件

```bash
nano openclaw.json
```

### 添加 QQ 渠道配置

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "你的App ID",
      "appSecret": "你的App Secret",
      "token": "你的Token"
    }
  }
}
```

### 重启 Gateway

```bash
pm2 restart openclaw-gw
pm2 logs openclaw-gw --lines 20
```

> 如果使用 Webhook 模式，确保服务器有公网 IP 且防火墙已放行端口

## 第 5 页 · 验证消息收发

### 测试步骤

1. 将机器人添加到一个 QQ 群（需群管理员权限）
2. 在群内 **@机器人** 发送测试消息：

> @AI助手 你好

3. 等待 AI 回复
4. 尝试多轮对话
5. 测试不同类型的消息

### ✅ 成功标志

- @机器人后，AI 在几秒内回复
- 回复内容自然流畅

## 第 6 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| 审核未通过 | 检查应用信息是否完整规范；根据反馈修改后重新提交 |
| 群里不回复 | 确认机器人已添加到群；确认消息中 @了机器人 |
| 连接频繁断开 | 检查网络稳定性；确认 WebSocket 未被防火墙拦截 |
| 回复被平台拦截 | 确保回复不含敏感词；在 soul.md 中加入内容安全约束 |
