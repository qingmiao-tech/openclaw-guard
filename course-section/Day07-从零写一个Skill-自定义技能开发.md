## 第 2.5 页 · 讲师提示语（可直接口播）

- **开场认知**：写 Skill 不是做一个大工程，而是把你一段稳定、可复用的流程交给 AI。
- **实操前提醒**：先坚持一个原则：一个 Skill 只解决一个清晰问题，不要一开始就做成“大而全”。
- **卡点转场**：AI 不调用 Skill，大多数时候不是能力不够，而是 `SKILL.md` 里“什么时候该用我”写得不够清楚。
- **复盘收口**：这一节最有价值的产出，不是知识点，而是学员手里真的多了一个自己做出来、能跑的 Skill。

## 第 3 页 · 为什么要自己写 Skill？

### 三个理由

1. 🎯 社区技能**无法满足**所有个性化需求
2. 📦 将工作经验和流程**沉淀**为可复用的技能
3. 🚀 让 AI 按照**你的方式**做事

### 三种沉淀方式

| 类型 | 说明 | 难度 |
|------|------|------|
| 流程型（SOP） | 标准操作流程 | ⭐ 最容易 |
| 工具型 | 自动化脚本封装 | ⭐⭐ |
| 判断型 | 决策规则和逻辑 | ⭐⭐⭐ |

> 💡 建议从**流程型**起步，最容易上手！

## 第 4 页 · Skill 文件结构

### 目录结构

```
skills/
└── my-skill/
    ├── SKILL.md       # 技能定义文件（核心）
    ├── scripts/       # 可选：附带脚本
    └── README.md      # 技能说明（可选）
```

### `SKILL.md` 核心内容

| 字段 | 说明 |
|------|------|
| 技能名称 | 简洁明了，说明用途 |
| 技能描述 | 告诉 Agent 什么时候该用它 |
| 执行步骤 | 自然语言描述的操作流程 |
| 输出格式 | 执行后的输出格式要求 |
| metadata（可选） | 声明依赖的命令、环境变量、配置条件 |

---

## 第 4.5 页 · Skill 放哪儿，为什么有时不生效

### 三层存放路径（优先级从高到低）

| 层级 | 路径 | 适合场景 |
|------|------|----------|
| 工作区 Skills | `./skills/` | 当前项目 / 当前 Agent 专用，优先级最高 |
| 全局共享 Skills | `~/.openclaw/skills/` | 多个 Agent 共用的自定义 Skill |
| 内置 Skills | OpenClaw 安装自带 | 官方基础能力，优先级最低 |

> 同名 Skill 时，**高优先级路径会覆盖低优先级路径**。
> 所以“我明明改了文件，怎么效果不对”，很多时候不是内容写错，而是**你改的不是当前生效的那份**。

### 两个课堂上最常见的坑

1. **文件名写成 `skill.md` 小写** → 当前官方写法应使用 `SKILL.md`
2. **改完直接在旧会话里测** → Skill 列表通常在**新会话**里重新加载，退出旧会话再测更稳

## 第 5 页 · Skill 编写规范

### 编写技巧

- ✅ 步骤要**具体、可执行**
- ✅ 描述要**明确**，避免模糊
- ✅ 写清楚**什么时候触发**、**做几步**、**输出成什么样**
- ✅ 如果依赖外部命令 / API / 浏览器，要提前声明
- ✅ 考虑**异常情况**处理
- ❌ 不要写太长，保持简洁

### 好的描述 vs 差的描述

| 差的描述 | 好的描述 |
|----------|----------|
| "搜索新闻" | "使用 Web 搜索工具，搜索最近 24 小时内的 AI 领域热点新闻，至少找到 5 条" |
| "整理内容" | "将搜索到的新闻按重要性排序，每条提取标题、来源和一句话摘要" |
| "输出结果" | "以 Markdown 列表格式输出，每条包含：序号、标题、来源、摘要" |

## 第 6 页 · 开发案例：每日新闻摘要 Skill

### 需求分析

- 🎯 目标：每天自动获取 AI 领域热点新闻，生成中文摘要
- 📥 输入：无（自动搜索）
- 📤 输出：结构化的中文新闻摘要

### 设计思路

```
Web 搜索获取新闻 → 筛选 AI 相关内容 → 生成中文摘要 → 固定格式输出
```

---

## 第 7 页 · `SKILL.md` 编写示例

```markdown
# 每日 AI 新闻摘要

## 描述
当用户要求获取 AI 新闻、科技热点或每日资讯时，使用此技能。

## 执行步骤
1. 使用 Web 搜索工具，搜索"AI 人工智能 最新新闻"
   和"artificial intelligence news today"
2. 从搜索结果中筛选最近 24 小时内的热点内容
3. 选取最重要的 5-8 条新闻
4. 对每条新闻提取：标题、来源、一句话中文摘要
5. 按重要性排序

## 输出格式
📰 AI 热点日报 | {日期}

1. **{标题}** - {来源}
   {一句话摘要}

2. ...（重复以上格式）

---
以上资讯由 AI 助手自动整理，仅供参考。
```

## 第 8 页 · 调试方法

### 调试四步法

1. **先看位置和文件名**：确认你改的是当前生效的那份 `SKILL.md`
2. **开新会话再测试**：避免旧会话继续使用旧的 Skill 快照
3. **查看可用状态和日志**：确认当前会话真的看得到它
   ```bash
   openclaw skills list --eligible
   openclaw logs --follow
   ```
4. **检查依赖和权限**：确认 Skill 需要的命令、环境变量、浏览器或工具权限都已经准备好

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| AI 不调用你的 Skill | 描述不够明确 / 还在旧会话里测试 | 优化描述，加入触发关键词；退出旧会话后重新进入 |
| Skill 文件找不到 | 路径或文件名错误 | 确认路径为 `skills/<skill-name>/SKILL.md` |
| 改了文件但效果还是旧的 | 同名 Skill 被更高优先级路径覆盖 | 检查 `./skills/`、`~/.openclaw/skills/`、内置 Skills 是否有重名 |
| Skill 需要的命令 / API 不可用 | 缺少 `bins` / `env` / 配置条件 | 安装依赖、补环境变量、检查浏览器或工具配置 |
| 输出不稳定 | 步骤描述不够具体 | 把“搜索、筛选、排序、输出格式”写成更明确的动作 |

## 第 10 页 · 实操任务 1：创建自定义 Skill

### 操作步骤

1. 进入你的 OpenClaw 工作区根目录，确认里面有 `skills/` 文件夹
2. `mkdir -p skills/daily-news` → 创建技能目录
3. `nano skills/daily-news/SKILL.md` → 编写技能定义
4. 按照编写规范填写名称、描述、步骤、输出格式
5. 保存文件
6. 如果你本来就开着旧会话，先退出，再重新执行 `openclaw tui`
7. 测试输入："帮我生成今天的 AI 新闻摘要"
8. 如果仍未识别，先执行 `openclaw skills list --eligible`；必要时再 `openclaw gateway restart`

### ✅ 成功标志

- AI 识别并调用你的自定义 Skill
- 按照定义的步骤生成新闻摘要

---

## 第 11 页 · 实操任务 2：迭代优化

### 操作步骤

1. 分析第一次测试的输出，找出不满意的地方
2. `nano skills/daily-news/SKILL.md` → 调整步骤描述或输出格式
3. 优先重新开启一个新会话测试；如果仍不稳定，再执行 `openclaw skills list --eligible` 和 `openclaw gateway restart`
4. 重复 2-3 次迭代，直到输出满意

### 优化方向

- 输出格式不够美观 → 调整格式模板
- 新闻不够相关 → 优化搜索关键词
- 摘要质量不高 → 细化摘要要求
- 经常不触发 → 补充更明确的触发描述
- 依赖外部服务 → 把所需环境变量和工具前提写清楚

### ✅ 成功标志

- 经过迭代后，Skill 输出质量明显提升

## 第 12 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| AI 没有调用 Skill | 检查描述是否明确；确认路径为 `skills/<name>/SKILL.md`；退出旧会话后重试 |
| 改了 Skill 但没生效 | 检查是否有同名 Skill 覆盖；执行 `openclaw skills list --eligible`；必要时重启 Gateway |
| 执行结果不符预期 | 细化步骤描述；加入更明确的触发条件和输出要求 |
| Skill 依赖外部命令 / API | 检查依赖是否安装、环境变量是否已配置 |
| 多次迭代仍不理想 | 将复杂 Skill 拆成多个简单步骤，先保证单步稳定 |

---

## 第 13 页 · 🔒 安全提示

- **第三方 Skill 按“不受信任代码”对待**：尤其是带 `scripts/` 的 Skill，启用前先读内容
- **不要硬编码敏感信息**：API Key、密码等不要写进 `SKILL.md`，应走环境变量
- **高风险工具先收紧权限**：涉及文件系统、命令执行、浏览器登录时，优先在测试环境验证
- **环境变量注入是“本轮运行有效”**：不要误以为配置后会自动变成你全局 shell 的环境变量
