# 渠道与模型

渠道和模型是 OpenClaw 真正“开始工作”的前提，所以 Guard 把它们放在首页四条主路径里。

## 推荐先配模型，再接渠道

建议顺序：

1. 先配置至少一个可用模型
2. 再连接至少一个官方渠道
3. 最后做联通性和收发验证

原因很简单：如果模型还没准备好，即使渠道接进来了，后面的工作流也很容易在第一步就失败。

## 模型页要先完成什么

一个“够用”的首发配置至少包括：

- 一个可用的 Provider
- 一个清晰的主模型
- 基本的调用测试

Guard 的模型页更强调“推荐配置 + 高级设置”，普通用户先完成最小闭环就够了。

## 渠道页怎么理解

当前首发版优先支持官方渠道，目标是：

- 让普通用户先接入稳定可用的官方能力
- 不把自定义插件体系放到默认入口
- 为后续扩展保留空间，但不牺牲当前通用性

如果你是从旧版 UI 或内部增强版迁移过来，需要记住一点：

- 独立的自定义飞书页面已经不再作为首发默认能力暴露
- 渠道页保留官方飞书等正式入口
- 更灵活的自定义插件加载能力会作为下一阶段规划处理

## 配置完成后看哪里

当模型和渠道接好之后，建议回到首页确认：

- 可用性是否已经变绿
- 建议动作是否已经收敛
- 风险提示里是否还留有关键阻塞项

## 常见误区

### 只接了渠道，没配模型

表面上看入口已经连上，实际执行链路仍然会失败。

### 配了很多模型，但没有明确主模型

普通用户更容易在后续使用里碰到“到底当前走哪个模型”的困惑。

### 一开始就追求复杂自定义

首发阶段更推荐先用最小可用配置跑通，再逐步做高级扩展。
