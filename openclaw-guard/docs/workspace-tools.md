# 工作区工具

Guard 把角色、文件和搜索收在一起，是为了让日常使用更接近“管理工作区”，而不是“切很多后台页面”。

## 角色

角色页主要帮助你确认：

- 当前有哪些角色
- 默认角色是谁
- 工作区是否存在
- 关键文档是否齐全

## 文件

文件页统一处理两类内容：

- 受控文件
- 核心记忆

这也是为什么 Guard 后来不再把“记忆”强行维持成独立顶级页，而是合并进文件体系。

## 搜索

搜索页的价值不是“做一个大而全的全文检索”，而是帮你更快找到：

- 某个工作区配置
- 某个记忆文件
- 某段历史记录

## 后续扩展方向

后续版本会继续强化：

- 新 Workspace 自动识别
- 绘画 / 导出目录的保护建议
- 发现高价值目录但尚未纳入保护时给出提醒

首发阶段先把“已有工作区和关键文件管理好”作为第一优先级。
