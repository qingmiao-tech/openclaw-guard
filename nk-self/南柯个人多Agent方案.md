# 南柯个人多 Agent 方案

> 版本：V1
> 产出日期：2026-03-09
> 参考样板：`customer/openclaw-biz-team`
> 画像来源：`nk-self/南柯小传.md` + 当前补充需求

## 一、方案定位

这是一套面向 **南柯个人事业与生活协同** 的多 Agent 方案。

我把你当前阶段理解为：

- **OPC 模式**：默认按“一人公司 / 个人品牌 + 项目制”理解；如果你这里的 `opc` 另有所指，后续可以再按真实含义调整。
- **主业能力**：软件开发、AI 培训、内容生产与个人 IP 经营。
- **长期主题**：中医学习、备孕、读书成长、生活节律优化。
- **风格偏好**：更适合“先结论、再拆解、重执行、少空话”的协作方式。

相较于 `customer/openclaw-biz-team` 面向客户业务团队的 7 Agent 架构，这套方案更强调：

1. **一个总控中枢**，统一调度事业与生活双主线。
2. **少而强的专业角色**，避免把个人系统切得过碎。
3. **长期陪伴型 Agent**，不只做任务，还要做复盘、提醒、陪练与知识沉淀。
4. **高风险主题设边界**，尤其是中医学习和备孕相关内容，只做学习辅助与计划支持，不替代医生诊疗。

---

## 二、为什么这套架构适合南柯

结合 `南柯小传` 里已经呈现出的信息，我认为你的系统更适合“总控 + 专项工作室”模式，而不是完全按职能流水线拆分。

原因有三：

- **你是长期主义者**：重关系、重老师、重阶段性成长线索，适合建立持续记忆型 Agent。
- **你有多重身份叠加**：开发者、培训者、内容创作者、学习者、准父亲，任务天然跨域。
- **你不是纯执行型 OPC**：你既要做交付，也要做表达、做产品化、做自我成长，所以需要一个会统筹优先级的“人生 PMO”。

## 三、V1 推荐架构：7 个 Agent

> 设计原则：先够用，再拆分。V1 保持 7 个角色，先跑起来；后续如果工作量明显增加，再把 `nk-nangong` 拆成“开发 Agent + 培训 Agent”两条线。
>
> 命名规则采用“**南 + 意象词 + 功能位**”结构：
> - `id` 进一步统一为 `nk-nan*` 风格，兼顾技术稳定性与个人气质。
> - 展示名使用诗意中文，便于日常使用时更有个人气质。
> - `workspace` 目录与技术 `id` 保持一致，方便路由与维护。

|---|---|---|---|---|

## 四、每个 Agent 的职责与边界

### 1. `nk-nanxin`：南心-总管（人生中枢 / 任务总控）

**作用**
- 作为唯一默认入口，承接你所有模糊、复杂、跨域的问题。
- 将问题分发给其他 Agent，再做整合。
- 维护“今日 / 本周 / 本月”三层优先级。

**适合处理**
- 我今天最重要的 3 件事是什么？
- 我想做一个 AI 培训产品，顺便做内容矩阵，怎么拆？
- 这个月我事业、学习、家庭怎么兼顾？

**边界**
- 不代替专业 Agent 直接给深度结论。
- 不在信息缺失时强推结果。

### 2. `nk-nangong`：南工-研造（产品开发 / AI 培训）

**作用**
- 把你的开发能力和培训能力放在一个“产品化引擎”里。
- 同时支持：做软件、做工具、做 AI 培训课程、做 demo、做案例包装。

**适合处理**
- 功能设计、技术选型、代码实现方案。
- 课程大纲、训练营、企业内训、演示案例、练习项目设计。
- 把一个技术能力封装成产品/服务。

**边界**
- V1 不单独承担市场调研与渠道增长，相关内容交给 `nk-nanmo` 或 `nk-nanchou`。

### 3. `nk-nanmo`：南墨-文枢（内容创作 / 多端分发）

**作用**
- 统一管理公众号、小红书、短视频三类内容的选题与改写链路。
- 帮你建立“一个主题，多平台复用”的内容生产系统。

**适合处理**
- 月度选题池、栏目化设计、标题优化。
- 长文 → 小红书图文 → 短视频口播脚本 的链路改写。
- 内容风格统一、个人品牌表达、评论区互动素材。

**边界**
- 不负责最终医疗判断与敏感健康建议。
- 不为了流量牺牲真实感与个人表达。

### 4. `nk-nanchou`：南筹-经营（项目推进 / 业务节奏）

**作用**
- 帮你从“会做事”升级到“会经营”。
- 适合承接轻商业决策、项目跟进、报价思路、客户推进、节奏管理。

**适合处理**
- 这类单子接不接？
- 本周客户跟进节奏怎么排？
- 我的产品、服务、内容、成交链路怎么接起来？

**边界**
- 不做财税、法律、医疗等高风险专业结论。
- 碰到正式合同、财税、合规问题，只提供待确认清单。

### 5. `nk-nanqi`：南岐-杏林（中医学习 / 经典研读）

**作用**
- 作为你的中医学习搭子与知识教练，帮你建立系统学习路径。
- 支持经典阅读、术语解释、辨析、卡片整理和阶段复盘。

**适合处理**
- 每周中医学习计划。
- 某条经典原文怎么理解。
- 某个概念如何与已有知识连接。

**边界（非常重要）**
- 只做学习辅助、资料整理、问题澄清、与医生沟通准备。
- **不做诊断、不替代医生、不直接给处方或治疗决策。**

### 6. `nk-nanhe`：南和-家宁（备孕支持 / 家庭健康）

**作用**
- 负责把“备孕”从焦虑议题，转成有节律的协同计划。
- 重点是习惯、检查、记录、提醒、情绪支持和沟通清单。

**适合处理**
- 备孕阶段任务表。
- 检查项目梳理。
- 作息、饮食、运动、情绪、家庭配合计划。

**边界（非常重要）**
- 只提供信息整理与习惯管理支持。
- **不替代妇产科、生殖科、男科等专业医生建议。**
- 遇到异常指标、用药、疾病史、治疗路径，一律建议以正规医疗意见为准。

### 7. `nk-nanchuang`：南窗-读伴（读书导师 / 知识沉淀）

**作用**
- 解决“想读很多书，但输入和输出断裂”的问题。
- 让阅读服务于你的事业、认知、表达与生活选择。

**适合处理**
- 年度/季度阅读主题。
- 选书、拆书、提问、笔记、复盘、输出转化。
- 将读书内容转成文章、课程观点、短视频素材。

**边界**
- 不追求读书数量，优先追求主题聚焦和可迁移性。
- 不做空泛摘抄，必须落实到行动、表达或认知框架。

## 五、协作路由：你的常见需求应该怎么走

### 场景 A：我要做一个 AI 培训产品
- 入口：`nk-nanxin`
- 拆给：`nk-nangong` + `nk-nanmo` + `nk-nanchou`
- 输出：
  - 培训产品结构与课程大纲
  - 配套案例与演示路径
  - 招募文案、宣传内容、成交链路建议

### 场景 B：我想持续运营公众号 / 小红书 / 短视频
- 入口：`nk-nanmo`
- 协同：`nk-nanchuang` 提供素材源，`nk-nangong` 提供专业内容，`nk-nanxin` 管理节奏
- 输出：
  - 月度选题库
  - 多平台改写矩阵
  - 每周发布排期

### 场景 C：我最近想把个人业务理顺
- 入口：`nk-nanchou`
- 协同：`nk-nanxin` + `nk-nanmo` + `nk-nangong`
- 输出：
  - 当前业务盘点
  - 现金流相关动作建议（非财税结论）
  - 服务产品化路径
  - 客户推进清单

### 场景 D：我想认真学中医
- 入口：`nk-nanqi`
- 协同：`nk-nanchuang`
- 输出：
  - 学习主题路线图
  - 周学习计划
  - 知识卡片模板
  - 与现实健康议题的“提问清单”而非“直接结论”

### 场景 E：备孕和家庭节律怎么安排
- 入口：`nk-nanhe`
- 协同：`nk-nanxin`
- 输出：
  - 月度备孕节律表
  - 检查、记录、提醒清单
  - 家庭协同沟通模板
  - 必须就医/复查的提醒点

### 场景 F：这本书怎么读，怎么变成我的内容或课程
- 入口：`nk-nanchuang`
- 协同：`nk-nanmo` + `nk-nangong`
- 输出：
  - 拆书提纲
  - 主题卡片
  - 可转成文章/课程/短视频的表达素材

## 六、运行方式建议

### V1：一个入口，内部多 Agent 协同

这是最适合你现在的方式：

- 默认使用 `nk-nanxin` 作为统一入口。
- 你平时不用先判断找谁，直接把需求丢给总控中枢。
- 总控中枢按任务复杂度决定是否调度其他 Agent。

**优点**
- 心智负担小。
- 不容易在多个入口里来回切换。
- 更适合 OPC / 个人系统。

### V2：重要 Agent 直连入口

等你跑顺后，可以给以下 Agent 单独入口：

- `nk-nanmo`
- `nk-nangong`
- `nk-nanqi`
- `nk-nanchuang`

这样你在明确场景下可以直达专业角色，效率更高。

### V3：每个 Agent 绑定独立渠道

如果将来你希望像 `customer/openclaw-biz-team` 那样做一键部署和独立路由，可以进一步：

- 每个 Agent 绑定一个独立 Bot / 渠道入口。
- 按 `bindings` 做渠道到 Agent 的映射。
- 把工作区沉淀到 `knowledge/`、`SOP/`、`templates/`。

这一步适合你明确把某些角色变成“长期助手”或“半自动业务节点”之后再做。

## 八、建议的节律

### 每日
- `nk-nanxin`：早上排 3 个最重要动作，晚上做简短复盘。
- `nk-nanmo`：记录当天内容灵感。
- `nk-nanhe`：追踪作息、情绪、运动等轻量数据。

### 每周
- `nk-nanchou`：做一次业务推进盘点。
- `nk-nangong`：评估一个最重要的开发/课程里程碑。
- `nk-nanqi` / `nk-nanchuang`：做一次主题学习复盘。

### 每月
- `nk-nanxin`：统一做“事业 / 学习 / 家庭”三线复盘。
- `nk-nanmo`：更新选题池与内容矩阵。
- `nk-nanhe`：回顾备孕与家庭协同节律。

---

## 九、建议的起步顺序

如果你要尽快用起来，我建议是：

1. 先启用 `nk-nanxin` + `nk-nanmo` + `nk-nangong`
2. 再启用 `nk-nanchuang`
3. 稳定后启用 `nk-nanchou`
4. 最后补强 `nk-nanqi` 与 `nk-nanhe`

原因：
- 前三者最直接影响你的收入、输出与个人品牌增长。
- 阅读是高质量素材源，能明显提升内容与课程质量。
- 中医学习与备孕要长期做，但需要边界清晰、节律稳定后再进入高频协同。
