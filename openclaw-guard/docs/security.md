# 安全与防误触

Guard 的安全能力不是“把东西锁死”，而是尽量让边界更清楚、误触更少、恢复更有把握。

## 安全页分成三部分

### 安全检查

告诉你当前有没有明显风险，例如：

- 关键服务未就绪
- 配置存在高风险项
- 环境边界不合理

### 权限模式

通过不同安全预设，快速调整工具权限范围。

### 主机加固

给出和平台相关的建议，而不是一刀切地要求所有人都做同样的加固动作。

## 权限模式怎么选

推荐把它理解为“你希望这台机器现在更偏向什么风格”：

- 更开放：适合本机试验和快速验证
- 更收敛：适合长期运行和更严格边界

切换预设时，优先看两件事：

1. 它会限制什么
2. 如果不适合当前阶段，之后怎么恢复

## 为什么 Guard 强调“可解释”

很多安全功能的问题不在于“不够强”，而在于用户不敢点。

所以 Guard 的目标是：

- 先说清楚适合谁
- 再说明会影响什么
- 最后再给动作入口

## 加固建议的定位

主机加固目前更像一个“平台相关建议层”，而不是自动替你接管整台机器。

例如在 Windows 上，独立低权限账户更适合：

- 长期后台运行
- 共享机器
- 风险更高的生产场景

但它不应该被当成所有普通用户的默认强制步骤。

## 推荐实践

- 首发阶段先处理高风险失败项，不用一次性把所有建议全做完
- 对普通用户，优先使用推荐的权限模式，而不是一开始就走最激进的加固
- 做高风险改动前，先保存恢复点
- 遇到不确定的告警时，优先看说明和建议动作，再决定是否真的要改系统级配置
