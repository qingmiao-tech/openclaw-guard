## 第 3 页 · 定时任务的价值

### 什么是定时任务？

让 AI 在**指定时间自动执行**某件事，无需人工触发

### 典型场景

- ☀️ 每天早上推送新闻摘要
- 📧 定时检查邮箱并汇总
- 📊 每周生成工作周报
- 🖥️ 定时监控服务器状态
- 📚 每日英语学习推送

> 💡 定时任务 = 给 AI 设一个"闹钟"

## 第 4 页 · Cron 表达式基础

### 格式：`分 时 日 月 周`

```
 ┌───────── 分钟 (0-59)
 │ ┌─────── 小时 (0-23)
 │ │ ┌───── 日 (1-31)
 │ │ │ ┌─── 月 (1-12)
 │ │ │ │ ┌─ 星期 (0-7, 0和7都是周日)
 │ │ │ │ │
 * * * * *
```

### 常用示例

| Cron 表达式 | 含义 |
|------------|------|
| `0 8 * * *` | 每天早上 8:00 |
| `0 */2 * * *` | 每 2 小时 |
| `30 9 * * 1` | 每周一 9:30 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 |
| `0 10 * * 1-5` | 工作日每天 10:00 |

> 🔧 推荐工具：[crontab.guru](https://crontab.guru) 在线生成和验证

## 第 5 页 · OpenClaw Cron 配置

### 管理命令

```bash
# 先做一个 10 分钟后触发的一次性提醒
openclaw cron add \
  --name "Morning hello test" \
  --at "10m" \
  --session main \
  --system-event "给我发送一条早安问候，包含今天的日期和一句励志名言" \
  --wake now \
  --delete-after-run

# 查看已有任务
openclaw cron list

# 强制手动跑一次（课堂验收非常好用）
openclaw cron run <任务ID> --force

# 查看某个任务最近几次运行记录
openclaw cron runs --id <任务ID>
```

### 也可以通过对话设置

> "帮我设置一个每天早上 8 点的提醒"

AI 会自动帮你配置 Cron 任务！

## 第 5.5 页 · 第一条定时任务，不要上来就写“每天 8 点”

很多同学课堂上最容易卡在这里：

- 任务其实已经创建成功
- 但你写的是“明天早上 8 点”
- 结果今天课堂上根本验收不了

推荐课堂顺序：

1. **先用 `--at "10m"` 做一次性测试**
2. 任务能跑通后，再改成正式的 `--cron "0 8 * * *"`
3. 如果是重活、长文本、日报类任务，优先用 `--session isolated`

一句话记忆：

> **先验通，再排时间；先跑一次，再谈长期自动化。**

## 第 6 页 · 心跳机制（Heartbeat）

### 心跳 vs Cron

| 对比项 | 心跳（Heartbeat） | Cron |
|--------|-------------------|------|
| 触发方式 | 周期性自动执行 | 精确定时触发 |
| 配置文件 | `HEARTBEAT.md` + `agents.defaults.heartbeat` | `openclaw cron` 命令 |
| 默认频率 | 每 30 分钟 | 自定义 |
| 适用场景 | 后台巡检、记忆整理 | 定时推送、定时任务 |

### `HEARTBEAT.md`

定义心跳时要做的事情，如：
- 整理记忆
- 检查待办事项
- 清理临时文件

### 一个够用的判断法

- **想要“每天 9 点准时发”** → 用 Cron
- **想要“每隔一段时间看看有没有需要提醒我的事”** → 用 Heartbeat
- **想要“20 分钟后提醒我一下”** → 用 Cron 的 `--at`
- **想要“不要污染主对话历史”** → 用 `--session isolated`

## 第 6.5 页 · 主会话 vs 隔离会话：课堂一定要讲

| 运行方式 | 适合什么 | 课堂口径 |
|------|------|------|
| `--session main` | 提醒、跟当前聊天强相关的事件 | 让提醒回到主会话，像继续跟你说话 |
| `--session isolated` | 日报、周报、批量搜索、长内容生成 | 让它单独跑，不污染主会话历史 |

最常见的错误是：

- 明明只是一个 20 分钟后的提醒，却写成 `isolated`
- 明明是每天一次的长日报，却写成 `main`

结果就是：

- 要么上下文接不上
- 要么主会话被自动化记录刷得很乱

## 第 9 页 · 实操任务 1：第一个定时任务

### 操作步骤

1. 添加定时任务：
   ```bash
   openclaw cron add \
     --name "Morning hello test" \
     --at "10m" \
     --session main \
     --system-event "给我发送一条早安问候，包含今天的日期和一句励志名言" \
     --wake now \
     --delete-after-run
   ```
2. `openclaw cron list` → 确认添加成功
3. `openclaw gateway health` → 确认 Gateway 正常
4. 如课堂想立刻验收，可执行 `openclaw cron run <任务ID> --force`
5. 等待任务触发，检查是否收到 AI 消息
6. 确认正常后，再改成正式时间

### ✅ 成功标志

- 在设定的时间点，AI 自动发送问候消息
- 你知道为什么这个任务应该放在 `main` 而不是 `isolated`

## 第 10 页 · 实操任务 2：每日热点日报

### 操作步骤

1. 确认 Day 7 创建的新闻摘要 Skill 可用
2. `openclaw skills list --eligible` → 确认本会话真的能看到技能
2. 配置定时任务：
   ```bash
   openclaw cron add \
     --name "AI daily brief" \
     --cron "0 10 * * *" \
     --session isolated \
     --message "执行每日 AI 热点日报任务，搜索最新 AI 新闻并生成中文摘要，输出为清晰的 Markdown 日报"
   ```
3. 手动触发一次：`openclaw cron run <任务ID> --force`
4. 用 `openclaw cron runs --id <任务ID>` 看运行记录
5. 根据输出效果优化 Skill 和任务描述
6. 确认自动化工作流稳定运行

### ✅ 成功标志

- 定时任务按时触发
- 生成结构化的 AI 热点日报
- 你知道为什么日报类任务更适合 `--session isolated`

## 第 11 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| 定时任务没有触发 | 先跑 `openclaw gateway health`，再看 `openclaw cron list` |
| 不确定是任务没建成功还是没运行 | 直接 `openclaw cron run <任务ID> --force` |
| 任务触发但没有输出 | 用 `openclaw cron runs --id <任务ID>` 看最近运行记录 |
| 日报内容质量不高 | 优化 Skill 步骤描述；加入更具体的筛选要求 |
| 不知道该用 Heartbeat 还是 Cron | 精确定时用 Cron；周期性巡检用 Heartbeat |

## 第 12 页 · 🔒 安全提示

- **定时任务的资源消耗**：频繁触发会消耗 API 额度和服务器资源，合理设置频率
- **任务内容审查**：确保定时任务不会执行危险操作（如删除文件、修改系统配置）
- **长任务优先隔离**：日报、周报、批量搜索优先用 `--session isolated`，避免污染主会话

## 第 14 页 · 社区实战案例：16 个 Agent 运营 13 个平台

> 💬 来源：社区多 Agent 自媒体运营实践

### 定时任务 + 多 Agent = 自动化运营体系

下面来看一个真实的多 Agent 自媒体运营架构——三层 16 个 Agent，其中定时任务是核心驱动力：

**Cron 定时任务的实际应用**

| 时间 | 任务 | 执行 Agent |
|------|------|-----------|
| 每天 9:00 | 自动推送 5 个选题到 Telegram | 运营 Agent |
| 每天 9:30 | 视频号选题推送 | 视频 Agent |
| 每天 14:00 | 执行内容创作任务 | 内容 Agent |
| 每天 21:00 | X/Twitter 运营候选推送 | 增长 Agent |

### 关键经验

- **cron 定时任务是灵魂**：很多人用 AI 写文章是"想起来才用"，定时任务让这件事变成系统——每天 9 点选题推过来，你不得不面对它
- **"被推着走"比"主动去做"有效 10 倍**
- **子任务可以用便宜模型**：日常推送用 MiniMax-M2.1，费用只有 Opus 的 1/3

### 心跳机制的实战配置

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "2h",
        activeHours: { start: "08:00", end: "24:00" }
      }
    }
  }
}
```

从默认的每 30 分钟降到每 2 小时，只在早 8 点到晚 12 点运行。每天从 48 次心跳降到 8 次，节省大量资源。

> 💡 一个人睡觉的时候，不需要有人每 30 分钟戳一下看你醒没醒。合理配置定时任务的频率和时段，是省钱的关键。
