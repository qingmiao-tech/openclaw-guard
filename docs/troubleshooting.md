# 排障与诊断包

当系统出现异常时，Guard 希望你不需要先翻很多原始日志，而是先知道下一步怎么做。

## 常见排障顺序

### 1. 先看首页

首页会先告诉你：

- Guard Web 是否在运行
- Gateway 是否可达
- OpenClaw 是否就绪
- 模型和恢复是否缺关键配置

### 2. 再看运维

运维页适合确认服务层问题，例如：

- 后台是否启动成功
- Gateway 是否需要重启
- 当前端口和运行路径是什么

### 3. 再看日志

日志页适合确认：

- 启动失败
- RPC 报错
- 后台动作执行异常

### 4. 导出诊断包

如果你准备寻求帮助，优先导出脱敏诊断包。

## 登录前就卡住了，找不到初始化密码怎么办？

先在本机终端执行：

```bash
openclaw-guard auth show-password
```

如果你只是想先确认当前环境还保不保留这条可回看记录，可以先执行：

```bash
openclaw-guard auth status
```

注意：

- 这个命令只会在本机 CLI 中显示密码，不会通过网页返回。
- 如果你已经改过密码，请直接使用当前密码登录。
- 如果命令提示“没有可回看的初始化密码记录”，通常表示当前环境早于这项功能创建，或者初始化密码已经被替换。

## 诊断包包含什么

- 机器和运行时概览
- Guard Web / Gateway / OpenClaw 状态
- 备份与恢复概览
- 最近日志
- 安全检查摘要
- 渠道、模型和工作区摘要

## 诊断包不会暴露什么

- API Key / Token / Secret / Cookie / 密码原文
- 本机用户名原文
- 用户主目录等敏感路径原文

## 下载方式

首页的“求助与反馈”卡片里可以直接下载。

### HTTP API

```http
GET /api/support/diagnostics?download=1
```

## 提 Issue 的推荐信息

建议一起提供：

1. 诊断包
2. 你期望发生什么
3. 实际发生了什么
4. 最小可复现步骤
5. 如果有界面问题，再补截图

## 什么时候优先考虑恢复，而不是继续修

如果你满足下面任一情况，优先考虑“回到某个恢复点”会更高效：

- 刚做过高风险改动
- 最近更新后开始异常
- 当前状态已经明显混乱
- 你更在意“先恢复可用”，而不是立刻找根因
