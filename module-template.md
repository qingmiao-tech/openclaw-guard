# 模块设计模板

> **用途说明**
>
> 本文件是 OpenClaw 实战训练营课程设计文档的统一模块模板。后续编写所有 Learning_Module（Day 1-12 必修模块）和 Channel_Lab（选修渠道 Lab）时，**必须**以本模板为参考，确保所有模块格式一致、字段完整。
>
> **使用方法：**
>
> 1. 复制下方对应模板（核心模块模板 或 渠道 Lab 模板）
> 2. 将方括号 `[...]` 中的占位符替换为实际内容
> 3. 逐项填写，不得遗漏任何必填字段
> 4. 编写完成后，对照末尾的"模块自检清单"逐项检查

---

## 布鲁姆分类法动词参考

编写学习目标时，请根据认知层级选用以下动词，确保目标可衡量、可验证：

| 认知层级 | 推荐动词（中文） | 适用场景举例 |
|----------|------------------|-------------|
| 记忆 | 能够列举、能够识别 | 列举 OpenClaw 的核心组件 |
| 理解 | 能够解释、能够描述、能够区分 | 解释 Gateway 与 Agent 的关系 |
| 应用 | 能够安装、能够配置、能够使用 | 配置 soul.md 人格文件 |
| 分析 | 能够分析、能够排查、能够对比 | 分析 Skill 执行报错的原因 |
| 评价 | 能够评估、能够选择 | 评估不同渠道的适用场景 |
| 创造 | 能够独立完成、能够设计、能够编写 | 独立完成一个自定义 Skill 的开发 |

> **提示**：每个模块的学习目标应控制在 **1-3 个**，优先使用"应用"及以上层级的动词。基础层模块可适当使用"理解"层级。

## 模块自检清单

编写完成后，请逐项检查以下内容，确保模块质量：

### 核心模块（Learning_Module）自检

- [ ] 模块标题、编号、时长、所属层级均已填写
- [ ] 学习目标为 1-3 个，且每个目标使用了布鲁姆分类法动词
- [ ] 前置知识已明确列出依赖的模块或知识要求
- [ ] 教学内容大纲包含至少 2 个知识点
- [ ] 至少包含 1 个实操任务，且每个任务有操作步骤、预期输出、故障排查
- [ ] 验收标准至少 1 条，且为学员可自行验证的具体成果
- [ ] 安全提示已填写（如不适用已标注说明）
- [ ] FAQ 至少包含 1 个问题
- [ ] 延伸阅读已提供（可选但推荐）

### 渠道 Lab（Channel_Lab）自检

- [ ] 渠道简介已填写
- [ ] 前置准备包含账号要求和开发者配置/机器人创建的关键步骤
- [ ] 对接步骤包含通用流程引用和渠道特有配置
- [ ] 消息收发验证方法已描述，包含预期结果
- [ ] 常见问题排查至少包含 1 个问题及解决方案
