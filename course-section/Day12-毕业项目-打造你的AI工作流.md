## 第 3 页 · 什么是 AI 工作流？

### 定义

将多个 AI 能力**串联**起来，形成一个自动化的工作管线

### 工作流 vs 单一任务 vs AI 团队

| 对比项 | 单一任务 | 工作流（本节） | AI 团队（Day 13） |
|--------|----------|----------------|-------------------|
| Agent 数量 | 1 个 | 1 个 | 多个 |
| 范围 | 让 AI 做一件事 | 让 AI 自动完成一系列事 | 多个 AI 各司其职 |
| 触发 | 手动触发 | 自动触发 + 手动触发 | 多渠道独立触发 |
| 能力 | 单一能力 | 多能力串联 | 多角色协作 |
| 价值 | 节省一次时间 | 持续节省时间 | 替代一个团队 |

> 💡 Day 12 的重点是**一个 Agent 串联多种能力**形成工作流。
> 如果你想让多个 Agent 扮演不同角色、各自独立工作甚至互相协作，那是 Day 13 的内容。

### 好的工作流特征

- ✅ 解决真实痛点
- ✅ 流程清晰
- ✅ 容错性强
- ✅ 可持续运行

## 第 4 页 · 项目选题指南

### 选题原则

从日常工作/生活中找**痛点**，选择**重复性高、规则明确**的任务

### 推荐选题

> 💡 本模块聚焦**单 Agent 单工作流**——用一个 AI 助手串联多种能力完成一条自动化管线。
> 如果你想搭建多个 AI 角色各司其职的"AI 团队"，请看 Day 13 进阶课程。

| 方向 | 示例项目 | 涉及能力 | 难度 |
|------|----------|----------|------|
| 信息聚合 | 每日行业资讯推送 | Cron + Web + Skill | ⭐⭐ |
| 个人助理 | 智能日程管理提醒 | Cron + Memory | ⭐⭐ |
| 内容运营 | 自动化内容创作管线 | Skill + Cron + Memory + Browser | ⭐⭐⭐ |
| 数据监控 | 网站变更监控告警 | Cron + Browser | ⭐⭐⭐ |
| 学习助手 | 每日英语学习推送 | Cron + Memory + Skill | ⭐⭐ |
| 知识管理 | 自动整理笔记与生成周报 | Memory + Skill + 文件操作 | ⭐⭐ |

## 第 5 页 · 选题自检

### 三个问题

在确定选题前，问自己：

1. ❓ 这个任务是否**重复性高**？
2. ❓ 是否有**明确的输入输出**？
3. ❓ 能否用 OpenClaw **现有能力**实现？

### 一句话描述

> "我要做一个 ___，它能自动 ___"

例如：
- "我要做一个**行业资讯聚合器**，它能自动**每天收集 AI 新闻并推送到 Telegram**"
- "我要做一个**学习助手**，它能自动**每天推送英语单词并在晚上复习**"

## 第 6 页 · 项目设计方法论

### 四步法

```
第一步：需求分析
  → 明确问题、定义输入输出、列出所需能力
      ↓
第二步：流程设计
  → 画出流程图、确定每步工具、设计异常处理
      ↓
第三步：分步实现
  → 先跑通主路径、再加异常处理、最后配置自动化
      ↓
第四步：测试验证
  → 手动测试、端到端测试、异常场景测试
```

## 第 7 页 · 实战示例：每日资讯聚合推送

### 需求

每天早上 9 点自动收集 AI 热点新闻，推送到 Telegram

### 流程设计

```
Cron 定时触发 (09:00)
    ↓
调用"新闻收集" Skill
    ↓
Web 搜索获取最新资讯
    ↓
AI 筛选和摘要
    ↓
保存为本地文件（备份）
    ↓
通过 Telegram 推送给用户
```

### 涉及能力

Cron + Skill + Web 搜索 + Memory + 渠道推送

## 第 9 页 · 实操任务 1：选题与方案设计

### 操作步骤

1. 回顾推荐选题列表，选择一个方向
2. 写下一句话描述："我要做一个 ___，它能自动 ___"
3. 列出需要的 OpenClaw 能力
4. 画出工作流流程（文字列表即可）
5. 确定每个步骤的输入和输出
6. 可以让 AI 帮你完善方案

### ✅ 成功标志

- 有清晰的项目方案
- 包含目标、流程步骤、所需能力

---

## 第 10 页 · 实操任务 2：分步实现

### 操作步骤

1. **实现核心 Skill**：编写并测试项目需要的 Skill
2. **配置基础设施**：设置 Cron、配置渠道推送
3. **串联工作流**：手动触发测试完整流程
4. **逐步调试**：
   - 先测试每个步骤独立工作
   - 再测试步骤间的衔接
   - 最后测试端到端流程
5. **配置自动化**：确认手动测试通过后，配置 Cron
6. **观察运行**：让工作流自动运行 1-2 次

### ✅ 成功标志

- 工作流可自动运行并产出预期结果

## 第 11 页 · 实操任务 3：项目总结

### 操作步骤

1. 让 AI 帮你生成项目总结
2. 按四个维度自评：

| 维度 | 评估问题 |
|------|----------|
| 实用性 | 是否真正解决了一个实际问题？ |
| 完整性 | 工作流是否端到端跑通？ |
| 稳定性 | 能否持续稳定运行？ |
| 可扩展性 | 是否容易添加新功能？ |

3. 列出 2-3 个后续优化方向
4. （可选）将 Skill 发布到 ClawHub 社区

## 第 12 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| 不知道选什么题目 | 想想每天花时间最多的重复工作；问 AI 推荐 |
| 某个步骤执行失败 | 在 Skill 中加入错误处理；查看日志定位 |
| 工作流输出质量不稳定 | 优化 Skill 指令；利用 Memory 积累经验 |
| 不知道如何评估效果 | 从"是否减少了重复工作"角度出发 |

## 第 13 页 · 🔒 安全提示

- **工作流权限最小化**：只给必要的权限
- **定时任务频率控制**：避免过于频繁，浪费 API 额度
- **敏感数据处理**：确保数据不会被意外泄露
