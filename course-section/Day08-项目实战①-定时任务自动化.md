## 第 3 页 · 定时任务的价值

### 什么是定时任务？

让 AI 在**指定时间自动执行**某件事，无需人工触发

### 典型场景

- ☀️ 每天早上推送新闻摘要
- 📧 定时检查邮箱并汇总
- 📊 每周生成工作周报
- 🖥️ 定时监控服务器状态
- 📚 每日英语学习推送

> 💡 定时任务 = 给 AI 设一个"闹钟"

## 第 4 页 · Cron 表达式基础

### 格式：`分 时 日 月 周`

```
 ┌───────── 分钟 (0-59)
 │ ┌─────── 小时 (0-23)
 │ │ ┌───── 日 (1-31)
 │ │ │ ┌─── 月 (1-12)
 │ │ │ │ ┌─ 星期 (0-7, 0和7都是周日)
 │ │ │ │ │
 * * * * *
```

### 常用示例

| Cron 表达式 | 含义 |
|------------|------|
| `0 8 * * *` | 每天早上 8:00 |
| `0 */2 * * *` | 每 2 小时 |
| `30 9 * * 1` | 每周一 9:30 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 |
| `0 10 * * 1-5` | 工作日每天 10:00 |

> 🔧 推荐工具：[crontab.guru](https://crontab.guru) 在线生成和验证

## 第 5 页 · OpenClaw Cron 配置

### 管理命令

```bash
# 添加定时任务
openclaw cron add "0 8 * * *" "给我发送早安问候"

# 查看已有任务
openclaw cron list

# 删除定时任务
openclaw cron remove <任务ID>
```

## 第 6 页 · 心跳机制（Heartbeat）

### 心跳 vs Cron

| 对比项 | 心跳（Heartbeat） | Cron |
|--------|-------------------|------|
| 触发方式 | 周期性自动执行 | 精确定时触发 |
| 配置文件 | heartbeat.md | openclaw cron 命令 |
| 默认频率 | 每 30 分钟 | 自定义 |
| 适用场景 | 后台巡检、记忆整理 | 定时推送、定时任务 |

### heartbeat.md

定义心跳时要做的事情，如：
- 整理记忆
- 检查待办事项
- 清理临时文件

## 第 9 页 · 实操任务 1：第一个定时任务

### 操作步骤

1. 添加定时任务：
   ```bash
   openclaw cron add "0 8 * * *" "给我发送一条早安问候，包含今天的日期和一句励志名言"
   ```
2. `openclaw cron list` → 确认添加成功
3. 为了快速测试，设置一个几分钟后触发的任务
4. 等待任务触发，检查是否收到 AI 消息
5. 确认正常后，根据需要调整时间

### ✅ 成功标志

- 在设定的时间点，AI 自动发送问候消息

## 第 11 页 · 故障排查速查

| 问题 | 解决方案 |
|------|----------|
| 定时任务没有触发 | 确认 Gateway 在运行；检查 Cron 表达式 |
| 任务触发但没有输出 | 检查任务描述是否明确；查看 Agent 日志 |
| 日报内容质量不高 | 优化 Skill 步骤描述；加入更具体的筛选要求 |
| Cron 表达式不确定 | 使用 crontab.guru 在线验证 |

## 第 12 页 · 🔒 安全提示

- **定时任务的资源消耗**：频繁触发会消耗 API 额度和服务器资源，合理设置频率
- **任务内容审查**：确保定时任务不会执行危险操作（如删除文件、修改系统配置）

## 第 14 页 · 社区实战案例：16 个 Agent 运营 13 个平台

> 💬 来源：社区多 Agent 自媒体运营实践

### 定时任务 + 多 Agent = 自动化运营体系

下面来看一个真实的多 Agent 自媒体运营架构——三层 16 个 Agent，其中定时任务是核心驱动力：

**Cron 定时任务的实际应用**

| 时间 | 任务 | 执行 Agent |
|------|------|-----------|
| 每天 9:00 | 自动推送 5 个选题到 Telegram | 运营 Agent |
| 每天 9:30 | 视频号选题推送 | 视频 Agent |
| 每天 14:00 | 执行内容创作任务 | 内容 Agent |
| 每天 21:00 | X/Twitter 运营候选推送 | 增长 Agent |

### 关键经验

- **cron 定时任务是灵魂**：很多人用 AI 写文章是"想起来才用"，定时任务让这件事变成系统——每天 9 点选题推过来，你不得不面对它
- **"被推着走"比"主动去做"有效 10 倍**
- **子任务可以用便宜模型**：日常推送用 MiniMax-M2.1，费用只有 Opus 的 1/3

### 心跳机制的实战配置

```json
"heartbeat": {
  "intervalMinutes": 120,
  "activeHours": "08:00-24:00"
}
```

从默认的每 30 分钟降到每 2 小时，只在早 8 点到晚 12 点运行。每天从 48 次心跳降到 8 次，节省大量资源。

> 💡 一个人睡觉的时候，不需要有人每 30 分钟戳一下看你醒没醒。合理配置定时任务的频率和时段，是省钱的关键。
