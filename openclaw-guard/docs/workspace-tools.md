# 工作区工具

Guard 把一些“长期会积累价值的资产”集中在较稳定的工作区工具分组里，而不是让它们散落在不同入口。

## 角色

角色页适合管理长期角色设定、行为模板和执行边界。

如果你在 OpenClaw 里会长期维护一组角色，这里比直接在文件树里翻找更适合做观察和整理。

## 文件

文件页不是单纯的文件浏览器，它更像“工作区资产管理”入口。

它承接了几类高价值内容：

- 记忆类 Markdown
- 工作区里的核心文本资产
- 长期需要维护的配置或说明文件

Guard 目前已经把记忆作为文件页中的核心模式来表达，而不是再单独拆成一个平行入口。

## 搜索

当你不确定某个内容在哪个工作区、哪个记忆文件或哪个角色模板里时，搜索页会比手工翻目录高效得多。

## 会话与日志的关系

工作区工具更偏“长期资产”，会话和日志则更偏“当前运行态”：

- 文件 / 角色 / 搜索：关注长期可维护资产
- 会话 / 日志 / 通知：关注现在发生了什么

这样划分的好处是，普通用户不会因为页面太多而分不清哪些是“要保存的东西”，哪些是“只是观察当前状态”。

## 哪些内容适合纳入恢复范围

从产品角度看，更值得优先纳入保护的是：

- 记忆和角色
- 新建的 Workspace
- 绘画、导出和重要结果目录
- 自动化或需要长期保留的脚本与配置

如果你新建了一个高价值 Workspace，后续版本会继续加强提醒与推荐纳入能力。

## 推荐使用方式

1. 把高价值 Markdown 资产放到清晰的工作区结构里
2. 在较大调整前先做一次保存
3. 用搜索找跨文件信息，而不是手工来回翻
4. 对新增 Workspace 保持“尽早纳入保护”的习惯
