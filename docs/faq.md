# FAQ

## Guard 和 OpenClaw 是什么关系？

OpenClaw 是主体运行时，Guard 是围绕它做安装、运维、安全、防误触和恢复的一层控制台。

## 第一次启动的初始化密码在哪里看？

第一次启动 Guard Web 时，终端会直接打印初始化密码。

如果你没来得及记下，可以在同一台机器的终端执行：

```bash
openclaw-guard auth show-password
```

如果想先确认当前环境是否还保留这条可回看记录，可以先执行：

```bash
openclaw-guard auth status
```

这个命令只会在本机 CLI 中显示密码，不会通过网页返回。

## 为什么 `auth show-password` 可能提示无法回看？

通常有两种原因：

- 你已经把初始化密码改成了自己的密码，此时请直接使用当前密码登录。
- 当前环境早于“初始化密码本机留存”这项功能创建，因此没有可恢复的本地记录。

## 恢复之后，后续版本怎么接着走？

Guard 默认不会给你新建分支，也不会改写历史。恢复会在当前主线上追加一个 restore commit，后续版本继续接在这个节点后面。

## 我不懂 Git，也能用“备份与恢复”吗？

可以。默认视图会把 Git 细节藏到“高级 Git”里，你只需要理解“保存现在”“恢复到这个状态”“最近有没有上云”。

## Guard 会不会自动接管我机器上已有的 OpenClaw？

不会默认强行接管。Guard 会优先检测当前安装来源，并尽量沿原来源更新或修复。只有官方能力不可用或安装损坏时，才进入 Guard 的补救路径。

## 为什么首页不把所有页面都展开说明？

因为首发体验更强调“带路”，而不是把用户扔进一个复杂工具台。首页的任务是告诉你现在能不能用，以及下一步去哪里。

## 成本为什么没有放在主导航里？

当前成本能力本质上还是估算值，不适合作为正式账单视图展示，所以它被降为弱化信息，而不是主产品能力。

## 为什么没有默认开放自定义渠道插件？

公开首发阶段更重视通用性和稳定性。官方渠道优先保留，自定义插件能力会作为下一阶段扩展规划，而不是默认路径。

## Linux 是否全部发行版都支持？

不是。当前初始化链路重点覆盖 Debian/Ubuntu、Fedora/RHEL、Arch 三类主流发行版，其他系统需要更多手工步骤。
